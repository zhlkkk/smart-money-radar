# 数据信任层：置信度系统 + 数据方法论透明化

**日期**: 2026-04-02
**状态**: approved
**触发**: 用户质疑"数据准不准"，暴露出数据管线缺乏可信度量化和透明化机制

---

## 问题定义

Smart Money Radar 当前数据管线存在两个层面的问题：

1. **用户感知层**：用户无法判断一条告警有多可信——所有告警看起来权重相同，数据来源不透明，没有 disclaimer
2. **技术层**：数据管线各环节（Birdeye 评分、DexScreener 行情、Helius webhook）的可靠性参差不齐，缺乏量化评估和交叉验证

### 数据管线审计结论

| 环节 | 可靠性 | 关键风险 |
|------|--------|----------|
| Helius Webhook | 中 | 无 HMAC 签名、无持久化队列 |
| Birdeye 钱包评分 | 中 | 权重拍脑袋、无回测验证 |
| DexScreener 行情 | 低-中 | 1-5 分钟延迟、可被假池子污染、无缓存 |
| 链上 Authority 检查 | 高 | 仅检查权限状态，不验证价格 |
| Claude AI 摘要 | 低风险 | 失败返回空串，不会幻觉；但输入不准则输出不准 |

---

## 方案：信任层架构

短期用透明化建立用户信任，中期用技术手段提升真实准确性。

---

## 短期实现（本次范围）

### 1. 告警置信度系统

#### 分级定义

| 等级 | 标签 | 触发条件 |
|------|------|----------|
| 🟢 高 | `信号强度: 高` | 4 项全满：链上权限安全 + DexScreener 数据完整 + 流动性 > $50K + 钱包评分 Top 20%（已订阅钱包中） |
| 🟡 中 | `信号强度: 中` | 满足 2-3 项，或某项数据缺失但其余正常 |
| 🔴 低 | `信号强度: 低` | 仅满足 0-1 项，或存在明确风险因子 |

#### 计算逻辑

```
得分 = 0
if 链上权限安全（mint + freeze 已撤销）  → +30
if DexScreener 数据完整（流动性/FDV 非 null）→ +25
if 流动性 > $50K                           → +25
if 钱包评分 Top 20%（在已订阅钱包中排名前 20%）→ +20

高 ≥ 80 | 中 ≥ 45 | 低 < 45
```

#### 关键设计决策

- 置信度表示"数据完整度和可信度"，不等于投资建议
- 数据缺失 ≠ 低置信（DexScreener 超时但链上通过 → 中）
- 权重先上线拍脑袋版本，后续根据回测调整

### 2. Telegram 告警消息改造

改造前：
```
🐋 聪明钱买入 $TOKEN
流动性: $120K | FDV: $5M
AI: xxx
```

改造后：
```
🐋 聪明钱买入 $TOKEN
📊 信号强度: 高 🟢
流动性: $120K | FDV: $5M | 池龄: 3d
🔍 数据源: Helius → DexScreener → Claude
⚠️ 风险: 无（Mint/Freeze 已撤销）
AI: xxx
数据来自第三方 API，仅供参考，不构成投资建议
```

### 3. Dashboard 告警卡片置信度展示

告警卡片增加置信度标签，与 Telegram 推送保持一致：显示等级（高/中/低）+ 对应颜色。

### 4. Landing Page 数据方法论 Section

新增"数据管线透明化"section，内容分 5 个步骤展示：

1. **交易监听**（Helius）→ "我们看到什么"：链上真实发生的交易
2. **聪明钱识别**（Birdeye + 自研评分）→ "我们跟踪谁"：持续盈利的交易者
3. **行情数据**（DexScreener）→ "代币值不值得关注"：第三方市场数据
4. **安全检查**（Solana RPC）→ "代币安不安全"：链上不可伪造的事实
5. **AI 摘要**（Claude）→ "为什么值得关注"：AI 辅助判断

底部附局限性说明：
- 行情数据来自 DexScreener，可能有 1-5 分钟延迟
- 聪明钱评分基于历史表现，不代表未来收益
- 低流动性代币的数据可能不完整
- 所有信号仅供参考，不构成投资建议

语气原则：诚实但自信，技术人看到专业度，普通人看到透明度。

### 5. Dashboard 侧边栏入口

侧边栏加"数据说明"链接，锚点跳转到 Landing Page 的方法论 section。

## 中期技术改进路线图（不在本次范围）

| 优先级 | 改进项 | 目的 | 复杂度 |
|--------|--------|------|--------|
| P0 | Birdeye 评分回测 | 验证"聪明钱"标签的真实预测力 | 中 |
| P1 | DexScreener 本地缓存 | 减少单点故障，降低被限流风险 | 小 |
| P1 | Webhook HMAC 签名 | 替换当前弱验证，防伪造 | 小 |
| P2 | 链上价格交叉校验 | 用 AMM 池链上数据验证 DexScreener 价格 | 中 |
| P2 | 置信度权重自动调优 | 根据回测结果替代拍脑袋值 | 小 |

### 回测思路（P0）

核心问题：被标记为"聪明钱"的地址，历史买入后 24h/7d 胜率是多少？

- 拉这些地址过去 30 天的交易记录
- 追踪每笔买入后代币 24h 和 7d 价格变化
- 计算真实胜率和平均回报
- 结果可直接展示在方法论页面："我们跟踪的钱包过去 30 天平均胜率 XX%"
